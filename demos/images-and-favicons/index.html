Images

This page contains responsive image examples.

All examples are inside of <div> tags with a max-width of 1200px. All the images have their dimensions actually printed over them; the text is not coming from the HTML.

Potential Image Problems

All bitmap images - GIF, PNG, or JPG - have a fixed number of pixels per inch; this is known as an images resolution. Screen resolution used to always be 72ppi, and there was a one-to-one relationship between a pixel in CSS and a pixel of a device screen. However, device screens have gotten much better and have much denser displays. Nowadays you might have to provide different images to different devices using a media query for the DPR (or device pixel ratio).

Using <picture> for multiple image options

HTML 5 introduced the <picture> element, which acts as a container for multiple <source> items with a srcset attribute specifying a particular image and a media attribute listing media-query like options under which to display that image. The <picture> also contains a normal <img> link which is used for all non-specified media cases, or if the browser does not support the picture element.

This lists five separate media widths: 300, 600, 1200, 1600, and 1800 pixels wide, plus a "standard" image as a fallback.

<picture>
	<source srcset="img/300_225.png" media="(max-width: 300px)">
	<source srcset="img/600_450.png" media="(max-width: 600px)">
	<source srcset="img/1200_900.png" media="(max-width: 1200px)">
	<source srcset="img/1600_1200.png" media="(max-width: 1600px)">
	<source srcset="img/1800_1350.png" media="(max-width: 1800px)">
	<img src="img/2400_1800.png" alt="Skytrain">
</picture>

Media queries for resolution

Until about 10 years ago, web images were generally scaled to 72 pixels per inch, because all screens were only capable of displaying 72ppi. Modern screens are capable of much higher resolutions, which can lead to blurry or oddly-sized images on certain devices.

Using media queries, or a media-query-like syntax in your HTML, you can serve different images to different screen resolutions, listed in specific pixel per inch values, or relative numbers. In this case, if a user has a Mac "retina" screen, it will display the double size image; other screens will see the default, smaller size.

<picture>
	<source srcset="img/1200_900.png" media="(resolution: 1x)">
	<source srcset="img/2400_1800.png" media="(resolution: 2x)">
	<img src="img/1200_900.png" alt="Skytrain">
</picture>

The <figure> tag

To increase accessibility, and to provide semantic context for images and text used to describe them, HTML5 introduced the <figure> and <figcaption> tags. The figure element is a container for an image (or picture) tag and a caption to provide context for the reader.

The <img> alt attribute - required for valid HTML - is used to describe the image for screen reader purposes. The caption should be text that is useful to all users.

Notice that the <figure> tag is styled with a margin by default.

<figure>
	<img src="img/600_450.png" alt="Skytrain">
	<figcaption>An overhead shot from the Bangkok Skytrain.</figcaption>
</figure>

Fullscreen background images

Large, full-height and width images (often called a "hero" image) are a common web design theme. They created by assigning an image to the background of an HTML element, and are placed using a number of CSS background properties.

The height of the container will determine how large the image appears; using the value 100vh for the element's height will ensure that it takes up the entire screen height on the user's browser.
